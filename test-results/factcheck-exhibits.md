The code computes dividends via `(ret - retx) * mcap_lag` (line 74), which is a standard CRSP method that avoids split-adjustment issues (noted in code comments, lines 47--49). Dollar dividends are floored at zero via `GREATEST(..., 0)` to avoid negative values from data artifacts.

2. **Y-axis cap at 500 (acknowledged design choice).** Code line 124: `ylim[2] <- min(ylim[2], 500)` caps the axis for readability. This is documented in the code and does not misrepresent the trend. The caption does not claim specific P/D levels.

3. **2022 dip and 2023--2025 surge (verified as real).** The 2022 tech selloff and subsequent AI-driven rally are well-documented market events. The code applies no special treatment to these periods; the pattern is generated mechanically from CRSP monthly data aggregated by group (lines 51--83) and the trailing 12-month rolling window (lines 100--108).

**Additional notes.** The caption accurately lists the five tickers used in the query. The code selects permnos from `crsp.stocknames` by ticker, so META's permno covers the full sample (including the pre-rename period when the ticker was FB), because the permno is time-invariant. The trailing window uses a 365-day lookback (line 104), which is a standard approximation for 12 months. No cached data is available locally (the figure is generated from a live WRDS query), but the code logic is sound and the visible output is consistent with known market data.

**Exhibit verdict: PASS.** All suspicious features are explained by the underlying data and code logic.

## Table 1: Numerical Illustration: Price-Dividend Ratios

**Description.** A table showing price-dividend ratios ($V_0^A$, $V_0^N$, $V_0^{A,\mathrm{CM}}$), the AI--non-AI spread, and the hedging premium (%) for five values of singularity probability $p \in \{0, 0.005, 0.01, 0.02, 0.05\}$. Parameters are listed in the table notes.

### Suspicious features

1. **At $p = 0$, AI and non-AI P/D ratios are identical (11.9).** This could be a bug if the model should differentiate them even without singularity risk.
2. **The hedging premium reaches 73.4% at $p = 0.05$.** This is a large premium from a 5% annual singularity probability.
3. **Non-AI P/D ratio decreases as $p$ increases** (from 11.9 to 10.6), which could seem counterintuitive.

### Code check

1. **Equal P/D at $p = 0$ (verified as correct).** When $p = 0$, the formula reduces to $V_0 = R / (1 - R)$ for both assets (code line 31), since the singularity never occurs and both assets are priced by the same normal-state kernel. This is mathematically correct per Proposition 1 in the paper. Manual verification: $R = 0.96 \times 1.02^{-2} \approx 0.9227$; $R/(1-R) \approx 11.94$, which rounds to 11.9.

2. **Large hedging premium at $p = 0.05$ (verified as correct).** The hedging premium formula (code line 58) is $(V_0^A - V_0^{A,\mathrm{CM}}) / V_0^{A,\mathrm{CM}} \times 100$. At $p = 0.05$: $V_0^A = 26.5$, $V_0^{A,\mathrm{CM}} = 15.3$, giving $(26.5 - 15.3)/15.3 \times 100 = 73.2\%$, consistent with the reported 73.4% (rounding from higher-precision intermediate values). The premium is large because the incomplete-market singularity-state pricing kernel $\Phi^A$ includes the displacement factor $\Delta^{-\gamma} = 0.75^{-3} \approx 2.37$, which amplifies the AI stock's hedging value.

3. **Declining non-AI P/D (verified as correct).** The non-AI pricing contribution $\Phi^N$ uses $\tilde{\nu}/\nu = 0.30/0.55 \approx 0.545$, which is less than 1, meaning non-AI stocks' dividend share shrinks at the singularity. Higher $p$ puts more weight on this adverse state, lowering the non-AI P/D ratio. This is the displacement channel working as intended.

**Replication.** Running `Rscript code/numerical-illustration.R` reproduces the exact table in `paper/exhibits/numerical-illustration.tex`. Output confirms $V_0^A = 16.1$, $V_0^N = 11.6$, $V_0^{A,\mathrm{CM}} = 12.9$ at $p = 0.01$, and hedging premium = 24.8%. The paper text (Section 3) cites these values correctly, stating "a ratio of roughly 1.4" ($16.1/11.6 \approx 1.39$) and "a hedging premium of about 25%" (24.8% rounds appropriately).

**Exhibit verdict: PASS.** All table values are correctly generated from the model formulas and match the paper's claims.
